# Get started

SemicolonDesign 2021의 모든 디자인은 디자인 원칙에 따르도록합니다.

### Install

아래 Shell 명령어를 통해 SDS(Semicolon Design System)을 프로젝트에 추가해보세요 ❤️

`npm install --save @semicolondsm/ui`

`yarn add @semicolondsm/ui`

### Use

당신이 원하는 UI Components를 추가해보세요 🎉

`import { Button, Badge } from '@semicolondsm/ui';`

그리고 아래와 같이 사용해보세요

```jsx
import React, { Component } from 'react';

import { Button } from '@semicolondsm/ui';

const Example: React.FC = () => {
    return <Button>Hello, world!</Button>;
};
```

### 사용자 중심 디자인

서비스를 디자인 할 때 개발자 중심이 아닌 사용자 중심으로 합니다. 개발자가 고생한만큼 사용자의 고생은 줄어들 수 있습니다.

### 보편적인 디자인

독창성보다 보편성이 훨씬 더 우위를 차지합니다. 사용자가 학습할 필요 없으면서도 이해하기 쉬운 디자인을 하려면 보편적인 관례를 따라야 하기 때문입니다.

### 꼭 필요한, 최소한의 디자인

우리가 하는 모든 디자인이 꼭 필요한지 늘 고민합니다. 특히 모바일 앱에서는 늘 상기해야 하는 중요한 원칙입니다. 덤으로 데스크톱 사용성도 함께 좋아집니다.

### 일관성있는 디자인

서비스를 넘어 디자인 시스템의 요소를 일관성 있게 사용합니다. 이 덕분에 다양한 제품, 플랫폼이더라도 사용자는 서비스를 쉽게 이해할 수 있습니다. 그뿐만 아니라 일관성 있는 디자인은 효율적으로 제품을 만드는 디자인 시스템의 토대가 됩니다.
